
محرر نصوص HTML
المرحلة الاولى من هذا المشروع هي  إنشاء محرر نصوص بسيط لتنسيق نص  عن طريق كتابة النص أو لصقه داخل مربع و تصدير كملف HTML يعرض في المتصفح. و بأدوات تنسيق محدودة. و بقالب منسق بإستخدام CSS. بإستخدام JavaScript. 
المرحلة الثانية من تطوير المحرر إضافة مزايا أخرى مثل حجم الخط و نوع الخط، إضافة صورة،إضافة رابط و تصدير كملف للطباعة و حفظه كملف pdf. 

أولا: محرر نصوص بسيط
لتحقيق ذلك، سنستخدم خاصية contenteditable في HTML، وهي تسمح للمستخدم بالكتابة والتعديل مباشرة داخل عنصر معين، مع إضافة أزرار بسيطة للتحكم في التنسيق (مثل الخط العريض، المائل، واللون) باستخدام جافا سكريبت.
إليك الكود الكامل في ملف واحد:
HTML
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
<meta charset="UTF-8">
<style>
.container { width: 80%; margin: 20px auto; font-family: sans-serif; }

/* شريط الأدوات */
.toolbar { background: #f4f4f4; padding: 10px; border: 1px solid #ccc; border-bottom: none; }

/* منطقة التحرير */
#editor {
            border: 1px solid #ccc;
            min-height: 200px;
            padding: 15px;
            outline: none;
            background: #fff;
        }

button { padding: 5px 10px; cursor: pointer; }
    </style>
</head>
<body>

<div class="container">
<h2>محرر النصوص البسيط</h2>

<div class="toolbar">
<button onclick="format('bold')"><b>B</b></button>
<button onclick="format('italic')"><i>I</i></button>
<button onclick="format('underline')"><u>U</u></button>
<button onclick="format('fontSize', '5')">كبير</button>
<button onclick="changeColor()">تغيير اللون</button>
</div>

        ابدأ بالكتابة هنا...
</div>
</div>

<script>
// دالة لتنفيذ أوامر التنسيق

    }

شرح الأجزاء الأساسية:
1. contenteditable="true": هذه هي الخاصية الأهم؛ فهي تحول أي عنصر HTML (مثل div) إلى مربع نص يشبه معالجات النصوص (مثل Word).
2. document.execCommand: هي دالة في جافا سكريبت تقوم بتطبيق أوامر التنسيق (مثل جعل النص عريضاً أو مائلاً) على النص المحدد داخل العنصر القابل للتحرير.
3. التنسيق: استخدمنا CSS بسيطاً لإعطاء مظهر مرتب لشريط الأدوات ومنطقة الكتابة.

كيف تعمل ميزة التصدير؟
1. document.getElementById('editor').innerHTML: تقوم هذه التعليمة بجلب كل النص المنسق الموجود داخل المحرر مع وسوم الـ HTML الخاصة به (مثل <b> أو <span> للألوان).
2. Blob: نستخدم كائن Blob لتجميع المحتوى النصي وتحويله إلى صيغة ملف قابلة للتحميل.
3. رابط وهمي (a): نقوم بإنشاء عنصر <a> في الذاكرة وتعيين محتوى الـ Blob له، ثم نجعله يُحاكي ضغطة المستخدم لتحميل الملف (a.click()).
بهذه الطريقة، سيقوم المتصفح بتحميل ملف HTML يحتوي على النص الذي قمت بتنسيقه، ويمكنك فتحه لاحقاً في أي متصفح وسيظهر بنفس التنسيق.
ثالثا: دعم اللغة العربية 
لضمان أن الملف الصادر يدعم اللغة العربية بشكل صحيح عند فتحه في أي متصفح، يجب أن نقوم بتضمين "ترويسة" (Head) كاملة للملف، بحيث تتضمن meta charset="UTF-8" (التي تدعم الحروف العربية) واتجاه الصفحة dir="rtl".
في الكود السابق، قمنا بتصدير المحتوى الداخلي فقط (innerHTML). الآن سنقوم بتعديل دالة exportHTML لتغلف المحتوى بهيكل HTML كامل وداعم للغة العربية:
J
<script>
function exportHTML() {
const content = document.getElementById('editor').innerHTML;

// إنشاء هيكل ملف HTML كامل مع دعم العربية
const fullHTML = `
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
    <meta charset="UTF-8">
    <title>مستندي المنسق</title>
</head>
<body>
    ${content}
</body>
</html>`;

// إنشاء ملف وحفظه
const blob = new Blob([fullHTML], { type: 'text/html;charset=utf-8' });
const a = document.createElement('a');
        a.href = URL.createObjectURL(blob);
        a.download = 'document.html';
document.body.appendChild(a);
        a.click();
document.body.removeChild(a);
    }
</script>

لماذا هذا التعديل مهم؟
1. charset="UTF-8": بدون هذا الوسم، قد تظهر الحروف العربية على شكل رموز غريبة عند فتح الملف في بعض المتصفحات أو الأنظمة.
2. dir="rtl": يضمن أن اتجاه النص في الملف المصدّر سيكون من اليمين إلى اليسار، مما يحافظ على تنسيق الفقرات العربية كما كتبتها في المحرر.
3. lang="ar": يخبر المتصفح بأن محتوى الصفحة هو باللغة العربية، مما يساعد في تحسين طريقة عرض الخطوط.
نصيحة إضافية:
إذا أردت أن يحتفظ الملف بـ "تنسيق" الألوان والخطوط (الـ CSS) في الملف المصدّر، يمكنك إضافة وسم <style> داخل fullHTML بنفس التنسيقات التي استخدمتها في الصفحة الأصلية، هكذا:
JavaScript
const fullHTML = `
<!DOCTYPE html>
<html lang="ar" dir="rtl">
<head>
    <meta charset="UTF-8">
    <style>body { font-family: sans-serif; padding: 20px; }</style>
</head>
<body>
    ${content}
</body>
</html>`;

بهذه الطريقة، سيظل الملف الناتج يبدو احترافياً ومرتباً تماماً كما ظهر في المحرر .

توضيح الميزات الجديدة:
1. حجم الخط (fontSize): استخدمنا عنصر <select> للسماح للمستخدم باختيار أحجام مختلفة، حيث يتم إرسال القيمة (من 1 إلى 7) إلى دالة التنسيق.
2. إضافة رابط (createLink): تطلب من المستخدم إدخال رابط (URL) ثم تقوم بتحويل النص المحدد إلى رابط تشعبي.
3. إضافة صورة (insertImage): تطلب رابط الصورة (src) وتدرجها مباشرة في مكان مؤشر الكتابة.
4. إضافة جدول (insertHTML): بما أن execCommand لا تملك أمراً مباشراً لإنشاء جدول معقد، قمنا بإنشاء كود HTML الخاص بالجدول برمجياً وإدراجه في المحرر باستخدام insertHTML.
5. القوائم (insertOrderedList): تُستخدم لإنشاء قوائم مرقمة تلقائياً.
ملاحظة تقنية:
دالة document.execCommand تعتبر قديمة (Deprecated) من الناحية التقنية في المتصفحات الحديثة، ولكنها لا تزال تعمل بشكل ممتاز في كافة المتصفحات، وهي الأسهل والأسرع لبناء محررات نصوص بسيطة. إذا كنت تطور تطبيقاً ضخماً، قد ترغب لاحقاً في البحث عن مكتبات احترافية مثل Quill.js أو CKEditor.

ماذا تغير؟
1. القائمة المنسدلة للخطوط: استخدمنا الأمر fontName داخل دالة execCommand. عندما يختار المستخدم خطاً من القائمة، يتم تطبيق هذا التنسيق مباشرة على النص المحدد في المحرر.
2. خيارات الخطوط: وضعت قائمة بالخطوط التي تظهر بشكل جيد جداً في معظم أنظمة التشغيل (Windows و Mac) وتدعم الحروف العربية بشكل ممتاز (مثل تاهوما و Arial).
ملاحظة للمستقبل:
إذا أردت استخدام خطوط أكثر جمالية (مثل خط "Cairo" أو "Amiri" من Google Fonts)، ستحتاج إلى إضافة رابط الخط في قسم <head> وتعديل خيارات القائمة المنسدلة لتشمل اسم الخط الخاص بـ Google Fonts.

شرح الإضافات:
1. <input type="color">: هذه أداة مدمجة في المتصفح تفتح لوحة ألوان احترافية. بمجرد اختيار اللون، تقوم دالة format بتطبيقه على النص المحدد (foreColor للون الخط، و backColor للون التمييز خلف النص).
2. ميزة الـ PDF:
• عند النقر على زر "حفظ كـ PDF"، ستفتح نافذة الطباعة الخاصة بالمتصفح.
• في هذه النافذة، ستجد قائمة باسم "الطابعة" (Destination)، قم باختيار "حفظ بتنسيق PDF" (Save as PDF). هذه هي الطريقة الأكثر أماناً وموثوقية في المتصفحات الحديثة دون الحاجة لتحميل مكتبات معقدة.
